## 产品不变量

- `task_assignments` 是触发中心与教师端共用的唯一任务实例及当前状态事实。
- 当前任务目录只保留已确认的 `G01`–`G10` 固定成长任务和 5 个个性化改善任务；系统不得从历史目录恢复或自行发明其他任务。
- 新教师首次进入 `teachers` 时，任务触发中心必须幂等初始化 10 条 `G01`–`G10` assignment，初始状态统一为 `ASSIGNED`（未完成）。教师端服务只更新已有 G/P 任务的执行状态字段，不创建固定任务。双方不得修改对方拥有的字段。
- G01–G10 初始化只建立共享任务事实，不创建站内通知、提醒或其他投递意图。缺少或不足 10 条固定任务是系统内部 `TASK_BASELINE_INCOMPLETE`，不是外部 `SOURCE_MISSING`。
- 浏览器客户端不得直连 PostgreSQL；共享写入只能由受限服务端数据库角色执行。两个服务不得共用超级用户或同一写账号。
- 状态机、终态不可回退、固定任务唯一性、字段写权限和 `row_version` 必须由数据库约束或触发器落实，不能只依赖页面约定。
- 教师端不得写入分值、总分、毕业或金牌结论；固定成长积分由本系统读取合法完成状态后幂等结算。
- 任务状态历史与通知投递历史分别记录，不能把任务行已创建推断成通知已送达。
- Mock 规则不能冒充业务正式发布配置。
- 任务行写入共享表即表示任务事实已创建；通知和外部动作输出仍只能证明系统创建了投递意图，没有真实消费方回执时不能写成“已送达”或“已执行”。
- API Key、密码和会话 token 不得进入配置版本、数据库业务 payload、审计事件、输出中心或项目账本。

## 开发原则

- 先做可运行垂直切片，再扩页面数量。
- 领域规则放后端，前端不复制触发、计分或状态判断。
- 所有写命令必须有幂等键、版本和明确错误码。
- 新增接口或共享表写路径必须配最小测试；修复 Bug 先补回归测试。
- 不提交 `.env`、密钥、真实学生信息或生产数据。
- PostgreSQL 是本地持久化事实源；当前固定单 API 进程。Redis、Celery 和多 Worker 只在真实吞吐或投递需求成立后引入。
- 建表和变更只走 Alembic；运行库不灌 Mock 业务数据，任务目录与默认配置只由显式 Seed 创建，服务启动不能隐式灌数据。Mock fixture 仅限隔离测试。
- `.env.local` 只放敏感值且不得读取、打印或写入文档；非敏感策略优先走版本化配置中心。
- 修改页面前先看 `project-context/PRODUCT_AND_UI.md` 和
  `project-context/AI_CHANGE_MAP.md`；不得恢复已删除的“任务发布”页面。
