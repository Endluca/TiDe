# TIT 成长系统数据库表结构

状态：唯一当前文档  
数据库：PostgreSQL  
核验日期：2026-07-23

## 1. 当前边界

本文档只描述当前 ORM 与已落库 Schema，不再保留另一套“目标表”。

- 当前公司测试库为独立数据库 `tit_growth_test`，有 27 张业务表，另有 1 张 Alembic 迁移元数据表；Web App 使用受限角色 `tit_growth_app`，不是数据库所有者或超级用户。
- 任务域物理表只有 `task_templates` 和 `task_assignments`。`task_assignments` 是两端共用的任务实例及当前状态事实。
- 当前任务目录是 G01–G10 固定必修成长任务（合计 30 分）和 5 个个性化改善任务（不加分）。
- 课程原始行无损保存在 `source_records`，`lesson_facts` 只是可查询投影；个性化规则命中及物化结果由 `personalized_trigger_matches` 追溯。
- 任务写入会由数据库校验所有权、状态流转、终态、固定任务唯一性和乐观锁，并写入 `audit_events`；状态变化同时产生 `outbox_events`。
- `data_import_batches` 同时支持一次性基线 `MANUAL_BASELINE` 和未来日更 `API_DAILY`；当前真实站外日更接口仍未接入。
- 站内通知、提醒、运营 Case 和外部动作表已可供调试，但真实投递 Worker 尚未接入，不能把表中记录解读为已送达。

### 1.1 外部数据日更

当前已有统一批次、原始行、教师快照和课程投影结构。手工基线走 `MANUAL_BASELINE`；生产每日外部接口将走 `API_DAILY`，但拉取与调度尚未实现。

生产形态需要每天从外部接口获取教师、课程和相关证据，然后完成拉取、完整性校验、标准化、幂等落库和受影响教师重算。以下内容尚未实现，不是当前表：

- 外部接口鉴权、分页/游标、水位和迟到修正规则；
- 每日调度、重试、失败告警和补数/重放；
- 接口原始响应的页级追溯、完整性水位和修正链。

教师接口的新增教师写入必须复用当前初始化入口：`teachers` 首次出现一位教师时，在同一业务事务内幂等补齐 G01–G10；已有教师的日更只更新资料，不重复生成任务。

### 1.2 通用口径

- `timestamptz` 为带时区时间；`date` 只保留日粒度。
- `jsonb` 保存已有复合输入或快照；稳定查询字段仍使用类型化列。
- `NULL` 表示来源未提供或尚不确定；`0` 只表示明确的零值。
- `REAL / DERIVED_REAL / MOCK / MOCK_SIMULATION / MOCK_PROXY` 用于区分真实、真实派生和测试/代理数据。
- 当前积分列使用 `double precision`，本文按实际 Schema 记录，不写成尚未落库的精确小数类型。

## 3. 数据底座

### 3.1 `data_import_batches`

一次外部数据接入的批次头。教师快照、课程明细和投诉规则都使用这张表；相同文件摘要与 Sheet 不重复导入。

| 字段 | 类型/必填 | 含义 |
|---|---|---|
| `batch_id` | varchar，PK，必填 | 导入批次唯一 ID |
| `source_kind` | varchar，必填 | 数据集类型，如教师快照、课程基线或投诉规则 |
| `sync_mode` | varchar，必填 | `MANUAL_BASELINE / API_DAILY`，区分一次性基线和每日接口批次 |
| `source_system` | varchar，必填 | 来源系统或数据集名 |
| `source_filename` | varchar，必填 | 导入文件名 |
| `source_uri` | text，必填 | 本地来源路径；不应放密钥 |
| `source_sha256` | varchar，必填 | 文件内容摘要，用于幂等和追溯 |
| `source_sheet` | varchar，必填 | 工作表名 |
| `snapshot_label` | varchar，必填 | 运营可识别的快照标签 |
| `data_mode` | varchar，必填 | `REAL / MIXED`，标明整批数据的真实性 |
| `column_count` | integer，必填 | 原文件列数 |
| `row_count` | integer，必填 | 原文件数据行数 |
| `header` | jsonb，必填 | 原始列头快照 |
| `status` | varchar，必填 | `VALIDATED / COMPLETED / FAILED` |
| `imported_at` | timestamptz，必填 | 导入动作时间 |
| `payload` | jsonb，必填 | 批次校验结果和其他非稳定元数据 |
| `created_at` | timestamptz，必填 | 记录创建时间 |
| `updated_at` | timestamptz，必填 | 记录最后更新时间 |

### 3.2 `source_records`

外部数据的无损原始行。类型化业务表只是它的投影；原行不在投影过程中丢失。

| 字段 | 类型/必填 | 含义 |
|---|---|---|
| `source_record_id` | varchar，PK，必填 | 原始行唯一 ID |
| `batch_id` | varchar，FK，必填 | 所属 `data_import_batches` |
| `source_sheet` | varchar，必填 | 来源 Sheet；API 日更可写稳定的资源名 |
| `source_row_number` | integer，必填 | 批次内原始行号，与 Sheet 一起唯一 |
| `business_key` | varchar，必填 | 该行的业务关联键，如课程 ID |
| `teacher_id` | varchar，可空 | 可解析时的教师 ID，便于追溯受影响教师 |
| `lesson_id` | varchar，可空 | 可解析时的课程 ID |
| `occurred_at` | timestamptz，可空 | 来源提供可信时区时的业务发生时间；不明时为 `NULL` |
| `row_sha256` | varchar，必填 | 标准化原始行摘要，用于校验和追溯 |
| `raw_payload` | jsonb，必填 | 完整原始行；包含学员 ID 等敏感字段时只允许受限后端读取 |
| `created_at` | timestamptz，必填 | 原始行落库时间 |

| `raw_payload` | jsonb，必填 | 无损保留的原始宽表行 |

| `updated_at` | timestamptz，必填 | 快照最后更新时间 |

| `score_policy_sha256` | varchar，必填 | 计分配置内容摘要 |
| `peak_slot_cnt` | integer，必填 | Peak 可约 slots 数；供给任务只看该字段，达到 40 时完成 |

| `is_cpl_tesol` | boolean，可空 | 是否已完成 TESOL；`true` 完成，`false` 明确未完成，`NULL` 来源未提供 |

v6 运行时还会按教师关联共享任务与课程投诉事实：G01–G10 的 assignment 总数、
完成数和模板分值来自 `task_assignments + task_templates`；L0 投诉次数来自
`lesson_facts.complaint_level_rank`。这些实时证据不回写成教师快照中的默认值。

### 3.7 `personalized_trigger_matches`

个性化规则的命中事实。它将“什么证据命中了哪条规则”与最终产生的教师任务、运营 Case、通知或数据待补记录连起来。

| 字段 | 类型/必填 | 含义 |
|---|---|---|
| `trigger_match_id` | varchar，PK，必填 | 规则命中 ID |
| `trigger_code` | varchar，必填 | 稳定触发规则编码 |
| `rule_version` | varchar，必填 | 命中时的规则标识 |
| `teacher_id` | varchar，FK，必填 | 受影响教师 |

### 5.2 `task_assignments`

固定引用的 `task_templates.payload.score_value` 计入成长任务分；部分完成会立即累计，

写入 `teachers` 时由本系统幂等创建 10 条 `ASSIGNED` 固定任务；少于 10 条表示内部
`TASK_BASELINE_INCOMPLETE`，不能推断为教师未完成，也不是外部 `SOURCE_MISSING`。

| `completed_at` | timestamptz，可空 | 可信完成时间；只有 `COMPLETED` 可非空 |
| `created_at` | timestamptz，必填 | 实例创建时间 |
| `updated_at` | timestamptz，必填 | 最后合法更新时间，由数据库维护 |

当前数据库硬约束：

- G01–G10 必须为 `FIXED_GROWTH + TRIGGER_CENTER`；5 个 P 任务必须是 `PERSONALIZED_IMPROVEMENT + TRIGGER_CENTER`。
- 同一教师同一 G 任务只能有一条。已确认不存在复营或跨营期回补。
- G01–G10 从 `ASSIGNED` 开始；该状态表示“任务已建立、尚未完成”。固定任务初始化不创建 `notifications`、提醒或投递意图。
- 教师端受限角色没有 assignment 的 `INSERT / DELETE` 权限，只能按状态机更新已有行的状态五字段。
- 有 `due_at` 时，三个时区证据字段必须同时存在；无截止时间时必须同时为空。
- `FAILED / EXPIRED / WAIVED / CANCELLED` 必须有 `status_reason_code`。
- `COMPLETED / EXPIRED / WAIVED / CANCELLED` 为不可回退终态；`FAILED` 可按状态机进入重试或完成。
- assignment 禁止删除；标识、教师、任务编码、模板引用、Why、展示标题、证据快照、来源和幂等键在创建后不可修改。
