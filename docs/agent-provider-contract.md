# 受约束 Agent Provider 合同

## 当前状态

当前已有 5 个个性化改善模板，但课程触发全部由确定性规则完成，不调用模型。模型可配置不等于当前会调用模型。

Agent 只保留给未来的多信号候选排序。单一确定信号继续使用确定规则；没有服务端候选集时，Agent 不创建 `task_assignments`。

## 输入边界

只允许发送完成选择所需的最小字段：

- 教师内部稳定 ID；
- 已标准化的信号代码、严重级别和证据状态；
- 本次服务端生成的候选任务 / 策略 ID；
- 优先级、时限、人工门和任务数量限制。

禁止发送 API Key、数据库凭据、学生身份、原始投诉文本和无关个人信息。

## 输出边界

模型只能返回服务端候选 ID：

```json
{
  "selected_candidate_ids": ["candidate-id"]
}
```

服务端必须再次验证：

1. ID 来自本次候选集；
2. 模板和策略仍处于已发布状态；
3. 证据完整、教师一致、未重复且未超过数量限制；
4. 人工确认任务没有被绕过；
5. 模型没有请求直接计分、改资格、处罚或执行高风险动作。

任何一项不满足都拒绝结果。Agent 不能创作任务、阈值、动作类型或教师可见自由文案。

## Provider 与失败策略

- Provider、模型、启停和熔断由配置中心的 `AGENT_POLICY` 管理。
- `OPENAI_API_KEY` 只由进程环境或部署密钥注入；本地放在被 Git 忽略的 `backend/.env.local`。
- 未配置 Key、超时、限流、服务失败或输出不合法时，本次安全降级，不创建未经确定规则证明的任务。
- 高风险动作只能进入运营 Case 或待审批请求，Agent 不直接执行。

调试记录只保存模型、输入哈希、候选 ID、结构化结果、降级原因、耗时和用量，不保存密钥或不必要的敏感原文。
