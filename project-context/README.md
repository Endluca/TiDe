# TiDe 项目背景入口

这个文件夹服务两类读者：需要理解产品的业务方，以及准备修改代码的 AI。

## 核心目标

缩短新外教达到“稳定合格供给”的时间，同时不牺牲履约、课堂质量和规则公平性。

系统要让运营回答三个问题：

1. 当前整体培养效果如何？
2. 哪些教师或事项需要运营介入，为什么？
3. 每个任务、分数和资格结论能否回到明确证据？

## 系统负责什么

- 建立固定成长任务和个性化改善任务；
- 根据确定性数据规则创建任务、提醒和运营处置事项；
- 汇总教师积分、出营资格与金牌资格；
- 提供运营 Web App、任务事实、课程证据和审计记录。

教师收到任务后如何学习、提交和完成，由教师端产品负责。两端共用
`task_assignments`，教师端只更新已有任务的执行状态。

## 当前边界

当前是公司测试环境产品，不是生产上线系统。已经有真实教师宽表和课程基线快照，
但外部日更、教师端生产接入、真实通知送达、生产监控和备份恢复尚未闭合。
