# 产品与页面

## 用户

当前 Web App 的直接用户是运营。页面结构按经营者视角组织：先看整体，再看待办，
最后下钻到教师、课程和任务证据。

## 当前页面

| 页面 | 用途 |
|---|---|
| 经营总览 | 查看培养规模、课程覆盖、当前待办、任务和出营转化 |
| 待办处置 | 只处理当前未终结的“运营处置”事项 |
| 任务进展 | 按任务查看覆盖与完成情况，需要时再展开教师流水 |
| 触达记录 | 查看任务、提醒、通知和运营 Case 等系统输出事实 |
| 教师档案 | 查看单个教师的积分、资格、任务和证据 |
| 课程证据 | 按课程回查触发信号和原始业务字段 |
| 任务规则 | 查看当前 10 个固定成长任务和 5 个个性化改善任务 |
| 积分与门槛 | 查看当前计分公式和出营、金牌门槛 |
| 操作审计 | 回查关键配置和状态变化 |

## 已确认的交互约束

- 进入页面时不自动更新；只有用户主动点击更新按钮才读取最新数据。
- 不恢复“任务发布”页面。固定任务随新教师自动初始化，个性化任务由规则触发。
- 待办处置只显示未终结的运营处置，不混入教师任务、提醒或数据缺口。
- 任务进展默认只显示任务统计；教师明细默认收起，展开后分页读取。
- `MIXED` 的业务外显是“多来源数据”，不等于“证据缺失”。
- 不向运营用户展示 Mock、API 连接状态、测试账号等开发术语。
- 页面文案必须解释业务意义，不直接暴露内部字段名作为主要标题。
